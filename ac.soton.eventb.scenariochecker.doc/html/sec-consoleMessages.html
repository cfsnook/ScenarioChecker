5.1 Scenario Checker Console Messages

The Scenario Checker Console displays the following messages: (Placeholders shown in angle brackets <xxx> will be replaced with actual names etc.).

  • Checking <machine name> - when the scenario checker is first started or started after stopping.

  • Restarted - when the scenario checker is restarted without stopping.

  • Stopped - when the scenario checker is stopped.

  • Can not run until context is set up - if a step is attempted when the ProB setup operation is enabled.

  • saved <recording name> - after the scenario is saved.

  • mode changed to PLAYBACK - after the mode has been changed to playback.

  • mode changed to RECORDING - after the mode has been changed to recording.

  • BigStep - fired [ext] <event signature> when the Big Step button is pressed (or an event fired from the list of enabled external events) and the event is fired successfully. (Note that it is also possible to fire an internal event if the previous big step aborted. In which case, [ext] is replaced by [int]). After this message is displayed the following messages will be displayed indented as part of the Big Step run.

    • - fired [int] <event signature> when an internal event is successfully fired

    • - Big step ran to completion when no internal events are enabled

    • - Big step aborted due to deadlock when no events are enabled

    • - Big step aborted due to loop on <event signature> when the event shown has already been fired a number of times during this Big Step.

  • Small step - <event signature> - when the Small Step button is pressed and an event is fired.

  • Small step aborted - nothing enabled - when the Small Step button is pressed but no events are enabled.

  • Run For - Run terminated after <#done> of <#ticks> ticks due to lack of progress - when the Run For button is pressed but a lack of progress is reported.

  • Run For - Completed all <#done> ticks - when the Run For button is pressed and completed

The following exceptional error messages indicate that ProB has not behaved as expected:

  • The Scenario Checker aborted because ProB appears to have stopped animating the machine (i.e. sent null)

  • The Scenario Checker aborted because ProB appears to be animating a different machine: <machine name>

  • Exception in ProB while excecuting operation <event signature>
