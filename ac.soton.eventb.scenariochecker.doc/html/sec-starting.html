2.3 Starting

The Scenario Checker is started using the context menu of a machine as shown in Fig. 1. I.e. by right clicking on a machine in the Event-B navigator and selecting the Scenario Checker menu item. An alternative is to ensure that at least one of the scenario views is opened (e.g. by selecting the Scenario Checker perspective) and then using the start toolbar button of the ProB support plug-in. (This is the green running man shown circled in red in 1). In both cases, the ProB support plug-in will be used to launch the scenario checker as well as any other participating animation plugins that are opened representing the same machine. For example, in Fig. 1, the BMotion Studio visualisation and state-machine will also start animating if they are linked to the machine that is being started for the Scenario Checker.

The Scenario Checker should be stopped by selecting the machine again and using the red standing man toolbar button. If another machine is started via the ProB support plugin, any current animation will first be stopped.
